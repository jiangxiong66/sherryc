计算机科学

Computer Science（计算机科学）的人总是被别的系的人挤兑，说你们为什么要叫计算机科学系呢，因为你们不科学，所以才要在名字上硬加上科学两个字，以显示你们是有逼格的，搞的是科学，是很高端的东西。

计算机和科学两个词搭配在一起是感觉怪怪的，计算机是人发明出来的机器，怎么就科学了呢，人家汽车也是人发明出来的机器，就没有人说汽车科学是吧，只有车辆工程或汽修技术。

人们用科学这个词，往往是关于研究自然界的现象，这些现象是即使没有人类也是客观存在的，比如生物、物理、天文，这些东西我们感觉很科学，计算机怎么就科学了呢？

让我们回顾一下历史，三千年前的埃及，尼罗河周期性的洪水带来的泥土在河谷形成肥沃的农田，他们根据土地的面积大小来决定交给法老多少税收，那时的埃及人掌握了丈量土地计算面积的方法，他们发明了几何。从三千年后的今天来看，他们发明几何最大的意义并不是掌握了测量土地面积的工具，而是找到了描述点、线、面关系的方法，从此开始人类便能够系统探索关于空间的知识。

计算机包含有形的硬件和无形的程序软件，程序是抽象的东西，它是一系列硬件能执行的操作指令，计算机科学便是关于怎样用这些抽象的东西，程序、软件描述问题解决问题的学科。

计算机他爹和冯诺依曼体系

冯诺依曼，计算机之父，他并没有发明电脑，也没有制造电脑。二战时期美军让宾夕法尼亚大学制造一台计算炮弹弹道的计算器，这台机器叫ENIAC，冯诺依曼当时在造原子弹，也需要算大量的数据，所以就去宾大把ENIAC找来用了一下，差不多在同一时期，美国还有另外一台电子计算器叫Mark I，冯诺依曼也去试用了一把。

用完以后冯老作了一个总结，他说，计算机应该这样造，它应该采用二进制，它应该包含运算器、控制器、存储器和输入输出设备五个组成部分，最重要的一点，它应该储存程序

储存程序，这一点在当时大概只有外星人能想到，我们现在会觉得在电脑上点两下网易云音乐就可以放音乐，点两下QQ就可以上网和人聊天是很自然的事。其实这是一件很逆天的事情，一台机器有A功能，没给他换任何零件，只是告诉了它一串0和1写成的天书，啪，它就有B功能了，老师傅都惊呆了。让Mark I算10+100再算10*100可是要拔一堆电线再新插一堆电线才能完成这个华丽的转身。

CPU

CPU 是计算机的大脑，它控制计算机的运行和完成计算功能。CPU的行为是由存储在计算机中的程序代码决定的。例如一个程序可以控制计算机进行两个数相加的运算。通过一些列简单的指令，计算机可以完成很复杂的任务。

现代的CPU集成在一块很小的电路上，我们称之为芯片，只有手指甲大小，里面含有上亿个可以完成简单数学计算（算加法）和逻辑计算（算与或非）的计算单元

Intel Core i5 含有7.74亿个晶体管

Memory

计算机运行一个程序，首先需要有一个地方存储程序的指令代码，其次需要有一个地方存储运算的结果，这个地方就是主存储器，我们一般叫它内存。这是冯诺依曼1946年提出来的想法，现在我们的电脑还是这个样子。

Storage

最常用的Storage就是我们的硬盘，我们的程序、软件、游戏、小电影都存在硬盘里面。硬盘和内存都是存储设备，它们有什么不同呢？关了电脑内存里面存的东西就没了，但是硬盘断了电资料是不会丢失的。我们的电脑开机时它内部做的一件事是检查硬件是否正常，第二件事就是从硬盘上把操作系统程序拷贝到内存中运行执行，系统就起来了。在使用电脑的过程中，例如Office办公软件，你不使用这个软件的时候，它是存储在硬盘中的，你双击启动它，计算机才会把Office的程序从硬盘拷贝到内存中，然后CPU再一条一条执行内存中的Office软件指令。

I/O 设备

Input/Output deices，输入输出设备，最常用的输入设备就是键盘鼠标喽，不用多解释了。输出设备，最常用的有显示器、投影仪、打印机等。

算法

前面我们讨论了计算机的硬件和结构方面的知识，下面我们讲一讲计算机科学，如何用计算机解决问题。算法，我们可以简单理解为解决问题的方法，比如我们在上小学的时候都学过的列竖式算乘法，就是一个算法。为了更好的理解算法，我们来尝试一下给算法下个定义。算法要能解决问题，它应当要能够被准确的表达，没有歧义，让人们能够理解它的每一个步骤；其次，一个算法应当是有效的，里面每一个步骤都能被执行；最后，一个算法应当是有限的，在一定的步骤后它能停下来得到结果。

总结一下，一个算法应当满足以下三个要求：

  • Clearly and unambiguously defined
  • Effective, in the sense that its steps are excutable.
  • Finite, in the sense that it terminates after a bounded number of steps.

程序语言和编译

用计算机解决问题包含两个步骤，首先我们需要设计出解决这个问题的算法，这个过程我们称之为算法设计（algorithmic design）；然后我们还要把这个算法用计算机能够理解运行的程序语言写出来，这个过程称之为编码（coding）。

程序语言有很多，列如C、Python、Javascript等等，在这个系列文章中我们主要讨论C程序语言。C语言是一种高级程序语言，为什么叫高级程序语言呢？因为它可以在各种计算机上运行，比如苹果Mac、IBM PC、iPad、安卓手机等等，其实这些不同的计算机用的CPU不同，各自有各自的机器语言，机器语言是低级语言，安卓手机的机器语言在ipad上是运行不了的。为什么C语言可以在各种机器上运行呢？因为C语言代码在变成最终的可执行代码的过程中经历了一个编译的过程。编译的过程很复杂，通俗的解释，我们用C语言写的代码通常放在一个.c的文件中，让后我们使用编译器把编写好的.c文件转化成.exe的文件，这个.exe的文件就是机器可执行的文件。

C程序的结构

上面我们已经编写了我们的第一个C语言程序Hello World，并且在Dev C++中编译运行了该程序。这个程序的功能很简单，就是在屏幕上输出一行字“Hello world”。几乎每一本C语言的书都会以这个程序作为第一个示例程序，这是C语言界的一个传统，这个程序是C语言的发明者Brian Kernighan和Denis Ritchie在它们的著作《The C Programming Language》里展示的第一个示例程序，所以几乎所有后来编写的C语言的书籍都会使用这个示例程序向这两位发明者表示敬意。

把Hello world这个程序做为第一个示例程序还有另外几个很实际的原因，这个程序很短小，但所谓麻雀虽小五脏俱全，通过这个程序，读者可以弄清楚程序编码、编译、运行整个流程，读者在这个过程可以熟悉自己的计算机软件环境。通过这个程序，读者可以了解一个C程序的大致组成部分和结构。

下面让我们再来回顾一下Hello world这个程序，我们先来观察一下整个程序的结构。

整个Hello world 程序包含三个部分。

第一个部分是程序注释，C语言中/*和*/之间的内容为注释（Comment），注释部分在程序中是没有功能的，它不是给机器看的而是给人阅读的，注释可以方便别人理解你写的程序，也方便自己以后理解现在的程序。写好注释是造福别人也是造福自己，当你编写复杂的代码时，你要相信很可能过几天以后你自己也很难读懂你现在的代码，写好注释吧。

第二部分#include <iostream>是引入了一个名为iostream（Input Output Stream）的头文件，这个头文件是C语言中与输入输出相关的库，引入了这个头文件，我们便可以在程序中使用这个库里定义好的东西，例如main函数中用到了std::cout

第三部分有4行代码，这段代码定义了一个名为main的函数，这个函数的返回值类型是int(integer 整数)，main后面有一对小括号()，括号里面表示的是这个函数接受的参数，这对括号里面没有东西，是空的，表示这个main函数不需要接受参数。

小括号后面跟了一对花括号{}，花括号中间包含了两行代码，第一行代码std::cout << "Hello world";这句代码的作用就是把Hello world输出到电脑屏幕上

后面一行代码return 0;表示函数执行到此处返回一个整数0，函数运行结束。

一个计算两个数相加之和的程序

为了更进一步了解C语言，我们下面讨论一个比Hello world复杂一点点的程序，这个程序add2.cpp如下所示。这个程序要求使用者在键盘上输入两个数字，程序算出这个两个数字的和，然后输出到屏幕。把代码复制到Dev C++中编译运行一下吧！

/*
  File: add2.cpp
  This program read two numbers, add them together,
  and prints their sum.
*/

#include <iostream>

int main() {
  int n1, n2, total;
  std::cout << "This program adds two numbers." << std::endl;
  
  std::cout << "1st number: ";
  std::cin >> n1;
  
  std::cout << "2st number: ";
  std::cin >> n2;
  
  total = n1 + n2;
  std::cout << "The total is " << total << std::endl;
  return 0;
}

首先我们仔细阅读以下代码，发现这个程序有很多元素与Hello World是相似的，比如程序最开始的部分，注释；头文件引入#include <iostream>;以及main函数。这些都是Hello world中也包含的。

我们看看main函数中的内容，main函数第二行std::cout << "This program adds two numbers." << std::endl; std::cout是我们再Hello World中已经接触过的，它后面跟了一个字符串常量，再后面的std::endl是新出现的元素，它表示换行。

相比Hello World，这个程序引进了一些新的概念。

main函数的第1行int n1, n2;定义了两个变量n1和n2，这两个变量的类型是int整型 (integer)。

变量可以理解为装数据的盒子，我们不知道用户要输入两个什么数字，所以我们在程序中准备两个装数据的盒子n1 n2，用户一输入，我们就把输入数据装进装数据的盒子--变量里。

在main函数的第5行，std::cin >> n1;, std::cin 表示标准输入，在正常情况下我们的电脑上标准输入是从键盘上输入，程序执行到这一行，它会等待用户从键盘上输入一些内容，用户按下回车键后输入结束。std::cin 后面更了一个双箭头号>>,双箭头号后面是我们之前定义好的变量n1，它们表示把刚才键盘上输入的内容放进变量n1中（请把变量想象成一个盒子）。

在把用户第一个和第二个输入分别放进n1 n2里之后，main函数的第10行代码total = n1 + n2，它表示计算n1加n2的和，并把这个和放入total这个变量中（还是把它想象成一个装数据的盒子）。再往下面是一个输出语句std::cout << "The total is " << total << std::endl;输出结果。

add2.cpp 结构分析

上面一节我们分析了add2.cpp程序的各条语句，了解一门程序语言的语法细节是写出正确程序的必要条件，但是要成为一名优秀的程序员，还需要能够从语言的细节中跳出来，对程序有宏观的把握，这一节我们来分析一下这个程序的宏观结构。

add2可分为4个阶段：

  • 变量定义阶段
  • 数据输入阶段
  • 计算阶段
  • 数据输出阶段

程序add2的这种结构是非常有代表性的，往后我们接触到的很多程序都有与add2类似的结构。

数据类型

在不同的应用中，程序会涉及到各种类型的数据，例如add2计算两个整数的和，就用到了整型数据（integer）。在一些其他的应用中，可能还会用到其他类型的数据，例如求给出半径求圆的面积，会用到圆周率，根据不同的精度要求，可能会取pi=3.14或pi=3.14159，这里的pi带有小数，它不是整数类型。又例如你用到的文档处理软件，里面最重要的数据是一个个的字符。当我们开始编写功能越来越复杂的程序，我们会需要把各种各样的信息组织在一起，形成复杂的数据。

当我们使用一个数据，不管是整数、小数或是字符，我们需要让C的编译器知道这个数据的类型。总的来说，一种数据类型可以通过以下两方面性质来定义：

  • 值域
  • 支持的操作

例如整型可以取(... -2, -1, 0, 1, 2 ...)这些值，可以支持加减乘除的操作（+ - * /）。以后面对一些复杂的应用我们会需要定义特有的数据类型，现在，我们先介绍一下C语言本身为我们提供的几个数据类型。

  • int
  • double
  • string

int整型，在add2中我们已经有使用过整型了，在C语言中，整型的取值范围是与系统的环境相关的，在32位计算机上，int的范围为[-2147483648, 2147483647]，可以初略记忆下，在目前大多数计算机平台上C语言整型的取值范围约在正负二十亿之间。

double双精度浮点数，如果数据可能带有小数，则可选取double作为储存类型。

变量、常量和表达式

当我们写一个需要执行一些运算的程序的时候，我们就需要用到表达式，C语言的表达式与数学上的表达式是类似的。例如之前的add2程序，我们用到的n1 + n2就是一个表达式。在C语言中，表达式是由terms和operators（运算符）组成的。operator，例如加号+、乘号*，表示一种运算的符号。在表达式中，一个term可以是：

  • 常量：例如直接出现在代码中的数字，像0和3.14都是常量，还有我们之前用到过的字符串常量"Hello world!"，这些都是常量。
  • 变量：可以理解为程序中用来装数据的盒子，像add2中的n1、n2和total都是变量。
  • 函数调用
  • 被括号括起来的表达式

在C语言中，一个变量有三个重要的属性，变量的名字、变量的类型、变量的值。

变量的命名规则：

  • C语言的变量的第一个字符为下划线_或英文字母，后面可接数字、字母和下划线。
  • C语言的保留字不可以作为变量名。例如int double if else while for等，后面我们慢慢接触到更多的C语言关键字，C和C++语言的关键字有几十上百个，在此就不一一列举了，如果不小心把变量命名成了关键字，编译器在编译时会报错，发现有冲突给变量换一个名字即可。

变量命名的建议：

  • 长度不要过长，变量名长度过长难敲代码也难读代码，例如aVeryVeryVeryAdorableGirl
  • 直观易懂，通过变量名容易推断出变量的意义，例如username totalLength
  • 小驼峰命名，所谓小驼峰就是变量的首字母为小写字母或下划线，之后单词的首字母为大写，例如lovelyGirlSherry

运算和运算优先级

整型的运算

在C语言中，整型支持加、减、乘、除、取余这些基本运算。

  • +：加法运算
  • -：减法运算
  • *：乘法运算
  • /：除法运算。在C语言中，两个整型相除，其结果仍为整型，例如9/4，其结果为2，余数部分会被扔弃掉。
  • %：取余（取模）运算。C语言中整型还支持取模运算。整型A模整型BA%B，其结果仍为整型，例如9%4，其结果为1。浮点类型例如double没有取模运算。

浮点型的运算

在C语言中，浮点型（例如double）支持加、减、乘、除四则运算。其规则与数学上的四则运算相似。

运算的优先级

C语言中整型、浮点型的四则运算与数学上的四则运算有相同的优先级规则，有括号先计算括号里面的运算，先乘除，后加减，优先级相同则从左往右依次计算。（整型的取模运算优先级与乘除相同）

以后我们还会学习C语言中更多的运算，例如逻辑运算、位移运算，C语言运算的优先级层级非常复杂，如果需要编写复杂的表达式，最稳妥的做法是适当运用括号，确保表达式的运算顺序与预期相同。

类型转换

前面我们已经介绍了整型和浮点型的运算，整型与整型的运算结果依然为整型，浮点型与浮点型的运算结果依然为浮点型。

运算符与自动类型转换

如果一个运算符左边和右边的运算数类型不同会发生什么呢？例如2 + 1.5，在C语言中如果一个运算符的一边是int另一边是double，则int会被自动转换成double进行运算，变成2.0 + 1.5。double转成int可能会造成小数部分被截断丢失信息，而int转成double则信息不会丢失，所以运算符左右两边既有int又有double的情况下，C语言会自动将int的值转换成double的值进行运算，这个过程叫作类型提升。

赋值与自动类型转换

在赋值语句中，如果左右两边的类型不同，则C语言会自动将赋值符号=右边的值转换成赋值符左边变量类型的值。例如double total = 0等价于double total = 0.0，又例如int x = 1.99等价于int x = 1。

double类型的值转换成int类型的值时，C语言并不是以四舍五入的方式转换其值，而是直接将小数部分截断，只留下整数部分。

显式类型转换

C语言的自动类型转换也叫作隐式类型转换，我们还可以显式的进行强制类型转换，例如double ratio = (double)5 / 9;，(double) 5会将整型数5转换成成浮点型数5.0。

关于数组

我们在前一章的Zeo旅费表格习题中遇到一个问题，我们需要在程序中存储Zeo每一天的花费，Zeo的旅行时间为5天，我们可以用5个double型变量在储存Zeo每一天的花费，double cost1, cost2, cost3, cost4, cost5，如果Zeo的旅行时间是100天呢，当然我们可以用100个double型变量如法炮制。这个方法是可行的，不过写起来代码好像有点长，有没有更好的办法呢，C语言有没有为我们提供什么方法，让我们可以更方便地存储这样一列同类型变量呢？

有的，C语言提供了一种叫做数组的东西，它可以存储一列同类型的变量。

数组有两个特性：

  • 相似性。一个数组中所有的元素类型是相同的，例如一个整型的数组，它里面所有的元素都是整型的。

  • 有序性。数组中的元素是有序无间隔排列的。

声明一个数组：

例如声明一个长度为5的整型数组，int zeoTravelCost[5];，与声明一个变量类似，首先是数组的类型，然后是数组的名称，与声明变量不同的是，数组还要指明数组的长度。

声明一个整型变量可以理解为向计算机申请了一个装整型数据的盒子，声明一个长度为5的整型数组，可以想象成是向计算机申请了5个装整型数据的盒子，这5个盒子紧挨着排成一排，从左到右从0依次编号。第i个变量就叫做zeoTravelCost[i]，我们把这个i称之为下标（index）,第i个变量也叫做下标为i的变量。

int zeoTravelCost[5];声明了一个长度为5的整型数组，这个数组中包含zeoTravelCost[0], zeoTravelCost[1], zeoTravelCost[2], zeoTravelCost[3], zeoTravelCost[4]5个元素

内存模型

C语言的习惯影响了很多程序员，这些程序员有的人后来又发明了新的编程语言，所以目前大部分主流程序语言都能找出很多与C语言相似的地方，例如很多主流程序语言的数组下标都是从0开始的，为什么数组下标要从0开始而不是从1开始呢？这还要从计算机的内存开始说起。

我们之前说过一个变量就像是一个存储数据的盒子，这些盒子是放在哪里的呢，它们就是放在内存里的，我们可以把内存想象成一个放满盒子的仓库，一个个装数据的盒子整齐的放在里面并且按顺序编好了号，我们把这些编号称之为内存地址。如下图所示：

程序中的每一个变量都有自己的内存地址。在C语言中可以通过&获取变量的内存地址，例如一个变量int a;，&a就是a的内存地址。当我们声明一个长度为n的数组时，计算机会为我们分配n个内存地址连续的盒子，例如，我们声明一个长度为4的数组arr，计算机就会分配4个地址连续的盒子，arr则指向这四个盒子最前面的那个，可以不妨假设这个数组中4个元素的地址分别为：1000、 1001、 1002、 1003，arr则指向地址为1000的盒子，而这四个元素的地址则分别是：arr + 0、arr + 1、arr + 2、arr + 3。所以C语言数组的下标是从0开始的而不是从1开始的。

在C语言中arr[i]与i[arr]是等价的，例如arr[1]写成1[arr]也没有错（不建议这么写哦，很多程序员可不知道有这么回事），因为编译器是把arr[i]翻译成*(arr + i)，把i[arr]翻译成*(i + arr)，而arr + i与i + arr的计算结果是一样的。
